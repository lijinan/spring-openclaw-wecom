# 目的
我想实现一个功能，使用springboot相关框架，搭建一个微服务，用于转发企业微信机器人消息到本地openclaw。

# 目录解释
openclaw ，是执行任务AI终端
sunnoy-wecom 是openclaw的一个企业微信插件
spring-openclaw-wecom 是一个springboot项目，用于转发企业微信机器人消息到本地openclaw

# 现状
在本地安装openclaw和sunnoy-wecom插件之后，没有办法直接配置到企业微信，因为企业微信需要配置公网的，并且是公司主体下认证的域名
也就是 localhost本地域名、tailscale转发域名、个人云服务器的转发域名，都不能使用

# 解决方案

## 实现内容

### 1. sunnoy-wecom 插件修改

新增文件：
- `remote-client.js` - WebSocket 客户端实现

修改文件：
- `index.js` - 添加远程模式支持
- `package.json` - 添加 ws 依赖，版本升级到 1.3.0

配置示例：
```json
{
  "channels": {
    "wecom": {
      "enabled": true,
      "token": "你的 Token",
      "encodingAesKey": "你的 EncodingAESKey",
      "remote": {
        "enabled": true,
        "serverUrl": "wss://company-server.com/ws/relay",
        "clientId": "my-openclaw-1",
        "reconnectInterval": 5000,
        "pingInterval": 30000
      }
    }
  }
}
```

### 2. spring-openclaw-wecom 中继服务

技术栈：JDK 1.8 + Spring Boot 2.7.x

项目结构：
```
spring-openclaw-wecom/
├── pom.xml
├── src/main/java/com/openclaw/wecom/
│   ├── WecomRelayApplication.java
│   ├── config/
│   │   ├── RelayConfig.java
│   │   └── WebSocketConfig.java
│   ├── controller/
│   │   ├── HealthController.java
│   │   └── WecomWebhookController.java
│   ├── model/
│   │   ├── ClientMessage.java
│   │   ├── ServerMessage.java
│   │   ├── VerifyPayload.java
│   │   └── WebhookPayload.java
│   ├── service/
│   │   └── PendingMessageManager.java
│   └── websocket/
│       └── RelayWebSocketHandler.java
└── src/main/resources/
    └── application.yml
```

端点说明：
- `GET/POST /webhooks/wecom` - 企业微信回调地址
- `WS /ws/relay` - WebSocket 连接地址（本地 OpenClaw 连接）
- `GET /api/health` - 健康检查
- `GET /api/status` - 状态查询

# 部署步骤

## 1. 部署中继服务

```bash
cd spring-openclaw-wecom
mvn clean package -DskipTests
java -jar target/spring-openclaw-wecom-1.0.0.jar
```

## 2. 配置企业微信

在企业微信管理后台配置：
- URL: `https://your-company-server.com/webhooks/wecom`
- Token: 与 OpenClaw 配置一致
- EncodingAESKey: 与 OpenClaw 配置一致

## 3. 配置本地 OpenClaw

在 `~/.openclaw/openclaw.json` 中添加远程模式配置。

## 4. 启动 OpenClaw

```bash
openclaw gateway run
```
